#### Atividade 1: Modelagem de Banco de Dados Relacional
**Período**: Semanas 1-4

**Caso de Uso**:
Você foi contratado por uma clínica médica que ainda usa fichas de papel. Eles atendem 200 pacientes/mês e têm 5 médicos de especialidades diferentes (clínico geral, cardiologista, dermatologista, pediatra, ortopedista). Precisam registrar:
- Dados do paciente (com histórico familiar de doenças)
- Agendamentos de consultas
- Prontuário de cada consulta (anamnese, diagnóstico, prescrição)
- Exames solicitados e resultados
- Vacinação

**Tarefa**:
1. Criar diagrama ER (Entidade-Relacionamento) completo

3. Criar índices apropriados para buscas comuns
4. Documentar decisões de modelagem

**Ferramentas**:

#### Atividade 3: API REST Básica - Sistema de Agendamento
**Período**: Semanas 9-12

**Caso de Uso**:
A recepção da clínica precisa de um sistema para agendar consultas. Eles recebem ligações de pacientes e precisam:
- Verificar horários disponíveis de cada médico
- Agendar consulta (validando conflitos de horário)
- Cancelar/remarcar consultas
- Ver agenda do dia de cada médico

**Tarefa**:
1. Criar API REST com ASP.NET Core Web API
2. Implementar endpoints:
   - `GET /medicos` - listar médicos e especialidades
   - `GET /medicos/{id}/agenda?data=YYYY-MM-DD` - horários disponíveis
   - `POST /agendamentos` - criar agendamento
   - `PUT /agendamentos/{id}` - remarcar
   - `DELETE /agendamentos/{id}` - cancelar
   - `GET /agendamentos?data=YYYY-MM-DD` - agenda do dia
3. Validações: não permitir agendamentos em horários ocupados, respeitar horário de funcionamento (8h-18h)
4. Conectar com PostgreSQL usando Entity Framework Core

**Documentação**:

### ATIVIDADES PRÁTICAS

#### Atividade 1: Sistema de Exames Laboratoriais com MongoDB
**Período**: Semanas 1-4

**Caso de Uso**:
A clínica agora oferece exames laboratoriais (hemograma, glicemia, colesterol, etc). Cada tipo de exame tem campos diferentes:
- Hemograma: contagem de hemácias, leucócitos, plaquetas, etc (múltiplos valores numéricos)
- Exame de imagem (Raio-X): referência ao arquivo de imagem, laudo textual do radiologista
- Urina tipo I: cor, aspecto, densidade, múltiplos componentes químicos
- Resultados chegam em formatos JSON variados de diferentes laboratórios parceiros

O sistema precisa:
- Armazenar resultados de exames de forma flexível (cada tipo tem estrutura diferente)
- Associar ao paciente e à consulta que solicitou
- Permitir anexar arquivos (PDFs, imagens)
- Histórico de alterações em resultados

**Tarefa**:
1. Modelar collections no MongoDB para exames
2. Implementar schema validation flexível
3. Criar sistema de referências (exame → paciente → consulta)
4. Simular armazenamento de arquivos (usar GridFS ou referências a S3/storage)

**Ferramentas**:
- [MongoDB](https://www.mongodb.com/docs/) - Documentação oficial
- [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) - Banco gratuito na nuvem
- [MongoDB.Driver](https://mongodb.github.io/mongo-csharp-driver/) - Driver oficial MongoDB para C#/.NET

**Repositório GitHub**: `clinica-exames-nosql`

**Checkpoint Mensal (30/04)**: Collections modeladas e exemplos de documentos

#### Atividade 3: API Completa de Exames com Integração
**Período**: Semanas 9-12

**Caso de Uso**:

4. Testes automatizados básicos (xUnit ou NUnit)
